# **MUST** have info version

The `info` object **MUST** have a `version` property that follows [semantic rules][1] to distinguish API versions.

> [!IMPORTANT]
> For openapi definitions marked with `info.x-api-type: pygeoapi`, this ruleâ€™s severity is automatically set to `warn` by the `override-severity-pygeoapi` rule. See [2].

## Invalid Example

```yaml
info:
  title: ...
  description: ...
  version: 1
  <...>
```

## Valid Example

```yaml
info:
  title: ...
  description: ...
  version: 1.1.0
  ...
```
